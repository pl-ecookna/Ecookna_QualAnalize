# Bug Fixes and Root Causes

## 2026-04-11 — PDF opening direction detection bug
- **Файл:** `bot/services/pdf_parser.py`
- **Проблема:** при парсинге PDF-файлов бот не определял `Открывание: Наружу` для позиций, где метка `Вид СНАРУЖИ на себя` находилась в первой ячейке строки таблицы, а не в секции формулы.
- **Причина:** табличный парсер проверял наличие `СНАРУЖИ`/`НАРУЖУ` только в тексте формулы, игнорируя полный текст строки таблицы.
- **Исправление:** расширена логика `_parse_text_by_tables` в `bot/services/pdf_parser.py` — теперь `is_oytside` определяется по полному тексту строки таблицы.
- **Проверка:** добавлен регрессионный тест в `tests/test_slip_formula_lookup.py` и пройдены `pytest tests/test_slip_formula_lookup.py -q`.
